Factory Method Pattern:
It defines an interface for creating an object, but lets subclasses decide which class to instantiate. Factory Method lets a class defer instantiation to subclasses.


Building a PizzaFactory. We can have following pizza stores.
  • NewYork Pizza Store
  • Chicago Pizza Store
So here above mentioned two are Creator.
Now each pizza store will provide the different types of pizza:
  • Cheese pizza
  • Veggie pizaa
and more..
So the different types of pizza(s) are our Product
Please Note: The cheese pizza of NewYork will not be same as Chicago.

Solutions & Design principles:

  1. We will use Factory Method pattern:
    Encapsulate the object creation. Let the subclasses decide which object to instantiate.
So Object Oriented Principles:
  • Encapsulate what varies
  • Favor composition over inheritance (HAS-A can be better than IS-A)
  • Program to interfaces, not to implementations
  • Strive for loosely coupled designs between objects that interact
  • Class should be open for extension but closed for modification
